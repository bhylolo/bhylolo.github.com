# 2017年7月28日
汉武帝刘彻，在4岁时被封为胶东王，刘彻小时候因为他母亲王娡王美人会作人，不但深得他父亲景帝刘启喜欢，连他姑姑馆陶公主刘嫖也很喜欢，喜欢到什么程度呢，刘彻并非长子，刘彻的母亲也非皇后，景帝之前有皇后，还立过一个太子，但最后都被废了，终立刘彻继承大统，他母亲王娡封后。

相传，在刘彻五六岁的时候，到他姑姑馆陶公主那儿去玩，馆陶公主让自己的女儿陈阿娇和一群宫女站在一起，就抱着刘彻问：这么多美女，你想找哪一个回去当老婆呢？
小刘彻说：如果能娶阿娇为妻，愿造一座金屋让她住。

——这就是“金屋藏娇”。

刘彻并不食言，在他7岁被立为太子后，就迎娶阿娇为太子妃，后来就是武帝朝的第一任皇后。但是，陈皇后阿娇的命不好，十年未生子嗣，武帝又不断的多情于其他美人，为了能争宠得子，于是阿娇就在后宫中行使了历来皇宫最大的忌讳————巫蛊。阿娇行使的是一种叫妇人媚道的巫术手段，后来被发现，被废后，囚居在“长门宫”，郁郁而终于此。

长门宫，本是阿娇的母亲馆陶公主谨献给汉武帝的一处园子。馆陶公主在丧夫之后，宠幸上府内一个男宠，叫董偃。两人之间的岁数差距，呃，基本上就像邓文迪和她的新进男友吧。如此这般，不可描述的好了多年，而且馆陶还很来事的将董偃介绍给皇城中的各个皇亲贵人，所以就弄得人尽皆知。但毕竟有流言飞语，且于国法不容，于是馆陶就把自己的长门园地献给武帝。武帝很高兴，竟然登门答谢。在家宴上，身为姑姑的馆陶亲自为武帝侍奉左右，待到恰当时机，叫出董偃。董偃头戴绿巾趴在殿下，武帝高兴的过去拉起他，称呼为“主人翁”。

——这就是“戴绿帽子”和“主人翁”。（另外，我一直怀疑，后人是借了刘嫖名字中的一个字，发明了一个词）

刘彻当上皇帝后，由于皇后阿娇没有子嗣，于是就让人有了计较。就像汉景帝的姐姐馆陶公主，不断向汉景帝进献美女，最终献的王娡上位，又将自己的女儿嫁给王娡的儿子，从而压中皇位，汉武帝也有一位好姐姐叫平阳公主。平阳公主在家中培养了很多美女。一日，武帝外出，平阳公主家中邀请款待。席间，武帝被一位歌女吸引，她就是卫子夫。武帝起身去上厕所，平阳公主立刻就让卫子夫前去服侍，在厕所里，武帝就临幸了卫子夫。走时，平阳公主将卫子夫赠予武帝。却不想，回到宫中，武帝就忘了卫子夫，冷落了一年都没召见过。等到某日，宫中清退一批宫女时，卫子夫也赫然在列。冥冥天意，武帝在审视清退宫女时，再次看见了卫子夫，于是又天雷地火，卫子夫怀上了儿子，至此，终于册封为卫夫人。卫子夫得宠幸之后，连同卫家外戚一同被重用，卫青是她弟弟，霍去病是她外甥。卫子夫生下儿子，在阿娇被废两年后，卫子夫立为皇后，儿子立为太子。以美色上位的总有一天会色衰而位危，在卫家权势盛极之时，遭人诬陷，太子起兵失败，卫子夫自杀于宫中，卫家遭灭门。卫子夫死后，汉武帝再也没立过皇后。

在卫子夫年老色衰之时，一日，乐师李延年给武帝唱歌，
唱到：北国有佳人，绝世而独立，一顾倾人城，再顾倾人国，宁不知倾城与倾国，佳人难再得。
武帝问：世间真有此女子？
平阳公主在一旁笑道：就是李延年的妹妹呀！

李夫人为武帝生下一位儿子后没多久就死了，在弥留之际，汉武帝来看她，李夫人用被子把自己的脸蒙住，
说道：妾长期病卧，未经修饰，不可面上，恳请将儿子和兄弟托付给陛下。
武帝说：你让我见一面，我就给你的兄弟加官晋爵。
李夫人背过身去：加不加官在陛下，见不见面在臣妾。
武帝很不高兴的走了。走后，旁边人问，为什么不肯见皇帝最后一面？
李夫人说：以色取人，色衰而情弛，皇上看见我现在丑陋的面容一定心生厌恶，而我不见他，虽然留有遗憾，但在他心里始终是我最美的样子。
李夫人死后，武帝很怀念她，让人招魂，夜里真的影影绰绰见到李夫人出现了，武帝哀叹道：立而望之，偏何姗姗其来迟。一伸手影子就消失不见了。

——这就是“绝世佳人”、“倾国倾城”和“姗姗来迟”

武帝宠幸一个女人，就会提拔和重用她的亲戚。源自汉高帝刘邦的白马盟誓：非刘不王，非功不侯，有违此者，天下共诛。李夫人被宠幸，她的哥哥李广利就被派出去打仗，想追寻匈奴单于主力决战，立下战功。但手握重兵，作为主将的李广利深入大漠却并没有截道匈奴主力，单于却遇上了偏将李陵。李陵是飞将军李广之孙，带领五千步兵负责粮草，单于有两万骑兵却不能将其吃掉，震怒之下的匈奴人不断增兵至八万，面对十六倍于己的敌人，李陵还能且战且退，激战八昼夜，匈奴人已经精神崩溃，打算天亮即撤兵，是夜，李陵部中一名逃兵投降匈奴告知一个重要军情：李陵的步兵之所以能打，仰仗的是弓弩，但箭已经用完，而且李陵并没有援军。天亮之时，距边关仅百里的峡谷中，匈奴人从山上往下发起冲锋，全歼汉军，最后逃回关内不足十余人，李陵被生擒，后投降。在听说李陵投降消息，汉武帝大怒，朝议之上，史官司马迁为李陵辩护犯下死刑，为作史记，司马迁接受宫刑 。而李陵的母妻子弟遭到斩杀。之后，彻底投降匈奴，单于将公主赐嫁给李陵，终老于塞外。

苏武出塞被扣留于匈奴，李陵曾去劝降，但苏武不从，李陵敬其气节，赠匈奴女于苏武，苏武在塞外成婚有子。到汉匈修和之时，苏武返汉，看着妻子和腹中的孩子，一扭头就走了，临行之前说：如果是个儿子就叫苏通国，如果是女儿就随母姓吧。回国之后的苏武卷入政治纷争，自己被罢官，长子被斩杀。

千年之后，杨家将杨继业塞外征战，没有援军全军覆没之际，一头撞死在李陵墓前。

——这就是李陵、司马迁、苏武和杨继业的故事。

汉武帝在古稀之年，遇到一个十几岁的神奇女子，她双手拳握不曾展开，而武帝轻轻一掰，就打开她的双手，手心中还有一个如意勾。这个女人就是钩弋夫人，她的儿子就是钩弋子——汉昭帝刘弗陵。刘弗陵是武帝最小的儿子，钩弋夫人也是武帝最年轻的宠妃，武帝虽立了太子，却将钩弋夫人，以及其他所有宠妃全部杀掉之后才死。

武帝死后没有皇后，托孤大臣自改李夫人为后，合葬武帝。

有人问我，是不是每个花心的男人心中都有一个忘不掉的女人，我想，位极如汉武帝般，尚且卫夫人之后不再立，李夫人之后不再宠，况乎凡人，爱的再多，旁人总能轻易的排出一二三位次。千年之后，留下的只有爱或不爱，任何男人追求的大义与之相比都会黯淡。

# 2017年6月2日：我喜欢的设计
也算是回应一下冯大辉的[生活里那些糟糕的设计](https://mp.weixin.qq.com/s?__biz=MjM5ODIyMTE0MA==&mid=2650969593&idx=1&sn=44e6922b655e16bc6512d17a30f1a333&chksm=bd3831c28a4fb8d4978204ea85f5d23bb26793625658479043ed8cf1f248106cbdf6bd9dbc25&mpshare=1&scene=1&srcid=0602thMvgVwY0oTdi3fSKZBw&key=e2d576b70311fc59d69298a7c2cd5b76c91382bfb6546c4f7a5e17eb81d9518055d342de717f600ac40c0fc505b87fceb821a44dc84a713005c641ada939a2d1786b0aa17a0d8f52caa0c54dacdf5545&ascene=0&uin=MTE5Nzk3&devicetype=iMac+MacBookPro8%2C1+OSX+OSX+10.12.4+build(16E195)&version=12020610&nettype=WIFI&fontScale=100&pass_ticket=5OI9yBaHm6iCmMjbjTOh4aGWEnytu0TPAexUVmDevSA%3D)。
* **品味**我觉得自己在品味和审美方面感觉很迟钝，要是我来做这方面，估计要让人抓狂了——比如，给美女拍照。所以，如果我要在淘宝上卖钥匙包，我会相信女人的审美，遵从付钱的大爷，只要买的多，钱给的多，我管他什么恶趣呢。
>在中学的时候，有一件很喜欢的套头卫衣，但有个缺点是没有口袋装不了东西，在外面缝一个又太难看，于是就在衣服的里面缝了一个，而且领口大，掏起来也挺方便的。这件衣服被我妹妹看上了，她以为是衣服买来里面就有一个暗袋的，找遍商场也没找出第二件，后来居然把我衣服要去穿了。再后来，很多运动衫运动短裤的衬里，都会缝一个暗袋。**我想说的是，自己动手做小东西，不是很有趣吗？**
* **USB**，很多电气设计都优先考虑不对称设计，以区分不同的电气规范。嗯，我应该研究一下USB的演变史，这不比吐槽更有意义吗？
* **身份证的设计**确实有点问题，但是可能是和当时的制卡工艺流程有关。倒是可以用一些“不对称设计或打孔等”，来标记正反面和日期，以避免制卡的出错率。
>国内现在很流行一个词，叫`互联网思维`，这个词背后隐含了一个前提：`产品可以通过不断的迭代，来修正问题，满足需求。`试想一下，如果软件是通过光盘，或者烧在硬件内的，这个前提就不存在了，同时，软件设计、研发、测试、发布等流程都会有很大不同，变得非常保守。由于升级、改错的代价大，很多硬件、实物产品是做不到互联网产品的发布方式，这些产品的设计会更加保守、更加遵从经验惯例——苹果的产品也是经历早期一年一两个版本的迭代后才达到现在的境界，但身份证的设计是不可能允许这样的条件的。
* **星巴克墙上插座** 这个问题有点奇怪，电气设备的安装位置、尺寸都有规范，如果公共场所的插座是安装在地上，或者朝上安装在桌面上，会有安全隐患，比如，拖地、用餐时，水可能会留到插孔里导致短路。你需要问的是，为什么现在手机、电脑的插头都随意设计尺寸，为什么就只有苹果电脑的插头有这个问题，Thinkpad三星惠普dell的就没问题，我的MacBook Pro的充电器还带一个三相延长线，只是那根线太长了。
> 对于这个问题，就好比，**夫妻生活不和谐，到底是男人的原因，还是女人的原因？**，冯说，是插座的原因，不是位置长得不对，就是尺寸不match。
* **车站的自动扶梯**我去过的所有城市的地铁站都面临一个问题：高峰时人太多——大家赶紧下车出站啊，不然真的会被挤的掉进铁轨里的。这个不是胡说，上海十几年前就经常发生这种情况，然后新闻上报道一条，早高峰有人卧轨自杀。所以，现在上海的地铁火车站是按照单相引导。另外，没有盲道，确实显得不够`现代文明`，但是在大城市，盲人一个人坐地铁火车，还是挺危险的，最好有个人陪着吧，至于小城市呢，走走路，坐坐公交车吧。（我就遇到过盲人一个人坐公交车，全车的人都抖哗哗的，司机都要不会开车了，很难想象他要是坐地铁怎么办，尤其是在人民广场，那不要出人命哒）
* **衣服领子上的标签**，我一般是用剪刀拆了，就像新西服、新媳妇一定要自己破个口才放心。
> 上次遇到一个做工控系统的人，聊下来在感叹道，现在国内看上去软件业很发达，但是在工控系统领域，国产水平太low，因为软件系统设计师完全没有硬件设计的能力，电脑和互联网把中国人教育成了只会写电脑和手机上的程序，多几个传感器的系统，就没能力搞了。就连 京东的仓库，自动化程度也没多高级。当把整个服装产业链了解下来，会发现，里面能做的事情太多了。领子上的一个标签不值得多吐槽。（软件不好用，你会说，为什么程序员不用自己的软件呢。但是，做衣服的人也穿衣服的呀，他们能接受的，就不好奇吗？）

### 最后，我喜欢的设计
* 尊重传统，也注重实际，设计的产品不但要解决问题，要美观，还要努力朝着把它变成规范的方向去做，设计师就会谨慎很多。
* 满足需求也不很难，美观也不很难，易用也不很难，高效也不很难，但是要平衡好，却很难。
> 为了解决`最后一公里出行问题`，共享单车搞了那么大一摊事儿，还有折叠自行车什么的。但，如果人人都会滑板、轮滑，是不是最高效的解决方法，可惜大多数人不会，就像Emacs／Vim 之于其他编辑器。这就是高效与易用之间的平衡问题。

# 2017年6月1日：人工智能

所以AI领域，是数据采集重要，还是算法重要，这都没关系，甚至它将取代多少已有行业，会影响多少人的饭碗，也不是首要的问题，首要问题是，如果BAT赚了这些钱，应该回馈社会多少。

# 2017年5月28日：知识付费
`知识付费`这个词是从几个app听到的：`分答、得到、小秘圈`。在这三款产品中：
* `分答`只在新闻中看见过
>分答我没有下载，因为我问不出什么问题，或者我想问的问题，别人未必肯回答。
* `得到`只在罗辑思维中听到过
>其实，我觉得罗振宇（对的，不是[罗振玉](http://baike.baidu.com/link?url=1ghZf6T9Ms6k9AgqLhpqJ43EGuCfSJk5VLm2WFaeKw0vfyF-HTc9Gos2lyN7EQFRSxoFRxnopwOTOklZz9-UcXLuZfqdhAjjVElPHxwWRGEeJ54BcENQpq5uWZpWfpG-)）做`得到`，就是一位两年读了七百本书的猛人，还要请别人给自己的产品撑门面，这本身就有点荒谬。
* `小秘圈`加入了一个圈子，支付了￥50
>这是我唯一付了钱的产品，我是从冯大辉的[小道消息](http://mp.weixin.qq.com/s/M_cs-U5ukP7PDBczKcFY9A)知道这个App的，冯大辉的文字让我讨厌，但是[飞总](http://mp.weixin.qq.com/s/CIBgpaqvRdqw7pKQCotckQ)写的文章，真的是CS PhD才能写得出的。之所以把这二位放在一起品味，是因为他们二位都号称是搞数据库的。笔杆子和嘴皮子毕竟是有区别的，所以￥50给了飞总。

总结一下关于`知识付费`，
* 期望通过问答、消息、新闻、摘录、利用碎片时间来学习知识，对于常人是不太可能的。因为任何知识都是有体系的，任何事业都是有“套路”的。只有对于已经掌握了只是体系，通过这种碎片信息来完善认知。这就是我在学习C#时，看[《C#程序设计语言》](https://www.amazon.cn/C-%E7%A8%8B%E5%BA%8F%E8%AE%BE%E8%AE%A1%E8%AF%AD%E8%A8%80-%E6%B5%B7%E6%9D%B0%E5%B0%94%E6%96%AF%E4%BC%AF%E6%A0%BC/dp/B002VED5DW/ref=sr_1_1?s=books&ie=UTF8&qid=1496059341&sr=1-1)时的感受——期望看语言设计者的感悟来学习编程语言，是不可能的。
* 以上的`知识付费`类产品，基本上是利用人们对于名人的猎奇，和对于教育的焦虑。
* 评价这类产品的价值，与其看付费人数，不如看`复购率`
* 如果程序员要做内容创作者，我觉得写技术文章不是好的选择,英语世界里聚集了世界上太多的聪明人，我们写的再好，也不过是翻译。倒是可以写技术史。譬如，我熟悉的[李维](https://gordonliwei.wordpress.com/)，我读过他绝大部分的书，但是写的最好的，还是[《Borland传奇》](https://item.jd.com/12626125179.html)
* 在中国，长盛不衰的知识热点永远是：
    * 中医和养生
    * 历史和秘闻
    * 求职和创业

* Markdown是洋人发明的，适合英文写作，中文写作还是有点别扭。主要是以下：
    * 中文的标点符号与Markdown语法符号冲突。对于在中文输入法下键入Markdown的语法符号，仍然是中文字符，不能变成英文字符（这个我还没有在搜狗、微软拼音、苹果中文下找到解决方法）
    * 对于k`括号[]{}()`使用英文的符号，而且输入以后光标能自动跳转到括号中间（貌似，这个应该是编辑器支持的功能），虽然按方向键可以解决，但任何打字人，都会有感受，一旦进入状态后，都不希望手离开主键盘区，连按方向键都是打断思路。
* 各个博客产品的Markdown还不统一，么有统一的语法标准，对于在互联网上发布内容，是很影响信息的链接
